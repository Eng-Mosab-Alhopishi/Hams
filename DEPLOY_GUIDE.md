# دليل رفع التطبيقات الجاهزة (APKs) إلى GitHub
# Guide: Uploading APKs to GitHub Releases

بما أننا لا نرفع ملفات الـ APK الضخمة داخل "الكود" لضمان خفة المستودع، سنستخدم ميزة **Releases** في GitHub.

---

## 🛠️ الخطوات (بالترتيب):

### 1. تحديد مكان الملفات (Locate Files)
ملفات الـ APK التي قمنا ببنائها موجودة في هذا المسار على جهازك:
`e:\new\app_v1\build\app\outputs\flutter-apk\`
*   `app-arm64-v8a-release.apk` (لمعظم الهواتف الحديثة)
*   `app-armeabi-v7a-release.apk` (للهواتف الأقدم)
*   `app-x86_64-release.apk` (للمحاكيات وبعض الأجهزة الخاصة)

### 2. فتح المستودع على GitHub (Open Repository)
بعد أن تقوم برفع الكود (سأعطيك الأوامر في الخطوة القادمة)، افتح صفحة المستودع `hams` على متصفحك.

### 3. إنشاء "إصدار جديد" (Create a Release)
1. في العمود الأيمن، انقر على **"Create a new release"** (أو ابحث عن Releases).
2. اختر رقم الإصدار (Tag version) واكتب: `v1.0.0`.
3. اكتب عنواناً للإصدار: `Initial Release - Hams v1.0.0`.
4. في المربع المخصص للمرفقات (Attach binaries)، قم بسحب وإفلات ملفات الـ APK الثلاثة المذكورة أعلاه.
5. انقر على **"Publish release"**.

---

## ✨ النتيجة:
الآن، أي شخص يزور مشروعك سيرى الكود نظيفاً في الواجهة، وسيجد زر **Releases** لتحميل التطبيق مباشرة وتجربته على هاتفه.
